# kadai03_line
建設現場作業進捗管理

## ②課題内容（どんな作品か）
- 職人さんがスマホから使い慣れたLINE風のUIで作業報告を行うと、バックエンドでデータが一元化されるプロダクト。

- 現場のやり方を大きく変えることなく、施工管理者が毎日行う「何人もの職人からの報告を工程表や出来高表へ転記する」という二重入力の負担を解消します。

- 職人さんの報告をもとにリアルタイムで「ガントチャート風進捗バー」が生成され、進捗率が特定の節目（25%〜100%）に達すると、システムが自動でチャットに通知を飛ばします。

## ⑤工夫した点・こだわった点
- 気軽なやり取り：現場のやり方を否定するのではなく、職人にとって一番馴染みのある「LINE風チャット」をベースにすることで、入力のハードルを極限まで下げました。
- データ連携の自動化サイクル：Firebase Realtime Databaseに保存された報告データを、GAS（Google Apps Script）を通じてスプレッドシートへリアルタイム自動転記する仕組みを構築しました。

## ⑥難しかった点・次回トライしたいこと（又は機能）
- 非同期通信とデータ構造のズレ：スプレッドシート側の列を挿入・並び替えた際、GASの配列番号やチャットアプリから送るオブジェクトのキー（データの名前）にズレが生じ、一時的にスプレッドシートへの反映が止まってしまうバグの解決に苦労しました。

- 現状は誰でも書き込める状態のため、Firebase Authenticationを実装して職種や業者ごとのアカウント制限をかけたい。
- 写真管理アプリとのAPIで施工履歴を紐付けたい
